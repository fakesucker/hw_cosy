# 客服场景预文本规范化（Customer Service Pre-TN）

本文档说明 CosyVoice 在 **wetext** 之前执行的客服专用预 TN 逻辑，实现位置为：

- **代码**：`cosyvoice/utils/customer_service_tn.py`
- **接入**：`cosyvoice/cli/frontend.py` 的 `text_normalize()` 中，在 `contains_chinese(text)` 为真时、**ttsfrd / wetext 之前**调用 `apply_customer_service_tn(text)`。

## 设计目的

wetext 使用 ModelScope 上的 FST，对阿拉伯数字会做 tag + verbalize。在客服话术里，经常出现：

- 应 **逐位朗读** 的号码（手机、热线、尾号、验证码等），却被读成 **整数或混合大数**（如「五千两百一十三万…」）；
- 应读成 **「三百兆」** 的宽带档位 `300M`，却被读成 **「三零零 M」**；
- 行首 `<|spk_*|>` 与 wetext 的配合见 `frontend.py`（剥离 spk 标签后再做 TN）。

本模块在 wetext 之前把目标数字 **先替换为汉字**，使 wetext 不再对这段数字做错误 verbalize（无数字时 wetext 常直接透传）。

## 如何开启 / 关闭

| 方式 | 说明 |
|------|------|
| 开启 | `export COSYVOICE_CUSTOMER_SERVICE_TN=1`（或 `true` / `yes` / `on`） |
| 关闭 | 不设置该变量，或设为 `0` / `false` 等 |

未开启时，`apply_customer_service_tn` 立即返回原文。

## 读法约定（两种数字汉字映射）

| 模式 | 含义 | 典型场景 |
|------|------|----------|
| **电话式（yao_one=True）** | 数字 `1` 读 **「幺」**，其余 `0–9` 读零一二…九 | 手机、热线、验证码、工号（配置项）、拨打号码等 |
| **通用逐位（yao_one=False）** | 数字 `1` 读 **「一」** | 尾号、身份证后四位、分机、IVR 单键、邮编等 |

带宽档位使用 **基数词 + 单位**（如「三百兆」），见下表「宽带速率」。

## 规则执行顺序（重要）

规则按 **列表顺序** 依次对全文做 `re.sub`，**顺序敏感**：更具体、更易冲突的规则在前，兜底规则在后。

当前 `_all_rules()` 顺序为：

1. `_rules_broadband_rate` — 宽带 `M` / 带宽语境下的 `G`
2. `_rules_carrier_hotline` — 运营商 / 银行品牌 + 官方客服号
3. `_rules_special_lines` — 400 / 800 / 95xxx / 12xxx 公益短号
4. `_rules_mobile_11` — 带语境的 11 位手机号 + 「您是…号码」类
5. `_rules_tail_and_id` — 尾号、后四位、证件后四位等
6. `_rules_verify_order_ext` — 验证码、工单号、分机、邮编、坐席工号、IVR 按键
7. `_rules_dial_phrase` — 「拨打 / 致电 / 联系…」+ 5–12 位号码
8. `_rules_standalone_mobile_11` — **兜底**：孤立 11 位本网手机号 `1[3-9]\d{9}`

---

## 规则明细

### 1. 宽带速率（`_rules_broadband_rate`）

| 子规则 | 匹配意图 | 转换结果 |
|--------|----------|----------|
| `\d{1,4} + M/m` | 带宽档位，如 `300M`、`300M的带宽`、`300M在…` | **基数词 +「兆」**（如「三百兆」）；排除 `MHz`、`MB`、`Mbps`、`Mpx` 等后缀 |
| `\d{1,2} + G/g` | 仅当后面紧跟 **带宽相关词**（带宽、光纤、宽带、套餐、速率、网络、网速、提速等） | `1G`→「千兆」；`nG`→「n + 千兆」（避免误改「5G手机」） |

### 2. 品牌 + 官方热线（`_rules_carrier_hotline`）

「品牌名 + 可选标点空白 + 固定号码」，号码按 **电话式逐位** 读。

覆盖示例：中国移动 10086；联通 10010；电信 10000；广电 10099；工行/建行/农行/中行/交行/招行/邮储及光大、民生、兴业、中信、浦发、平安等 **955xx** 表内号码。

### 3. 特殊热线形态（`_rules_special_lines`）

| 类型 | 行为 |
|------|------|
| `400-xxxxxxx`（10 位，可含空格/横杠） | 整段逐位（幺） |
| `800-xxxxxxx` | 同上 |
| `95xxx`（5 位，非字母数字边界内） | 逐位（幺） |
| `12xxx`（5 位，如 12345、12315） | 逐位（幺） |

### 4. 11 位手机 — 语境匹配（`_rules_mobile_11`）

**A. 号码在「提示语」之后**（如「您的手机号是」「号码为」「本机…」等），中间允许「是/为」及标点空格，号码 `1[3-9]\d{9}` → 电话式逐位。

**B. 「您是 … 号码」**  
匹配：您是 / 请问您是 / 跟您确认您是 / 麻烦您确认您是 / 跟您核对您是 / 确认(一下)后接您是 / 核对(一下)后接您是 → 空白 → 11 位 → `号码`。防止 wetext 把中间 11 位读成大数。

### 5. 尾号与证件后几位（`_rules_tail_and_id`）

尾号、手机尾号、后四位、末四位、身份证/证件后四位、账号/卡号后四位等 + 3–8 位数字 → **逐位，1 读「一」**。

### 6. 验证码、工单、分机、邮编、工号、IVR（`_rules_verify_order_ext`）

| 场景 | 数字长度 / 说明 | 读法 |
|------|-----------------|------|
| 短信验证码、动态口令、校验码、各类密码 + 数字 | 4–8 位 | 电话式（幺） |
| 工单/订单/案件/受理单/业务/参考 + 号 + 数字 | 6–22 位 | 电话式（幺） |
| 分机 + 数字 | 2–5 位 | 逐位，1 读「一」 |
| 邮编 / 邮政编码 + 6 位 | 6 位 | 逐位，1 读「一」 |
| 坐席/客服/工号/员工编号/服务专员 + 数字 | 3–8 位 | 电话式（幺） |
| 请按 / 按 / 选择 + 单键 `0-9` / `*` / `#` | 单字符 | `*`→星号，`#`→井号；数字逐位 1 读「一」 |

### 7. 拨打类短语（`_rules_dial_phrase`）

「请(可)拨打 / 致电 / 联系 / 咨询 / 呼叫 …」+ **5–12 位连续数字**（长度限定为常见热线/固话/手机长度），整段数字 → 电话式逐位。

### 8. 孤立 11 位手机 — 兜底（`_rules_standalone_mobile_11`）

在 **非字母数字边界** 内的 `1[3-9]\d{9}`（大陆常见号段），若未被上文规则改写，则整段 → 电话式逐位。  
**不**匹配第二位为 `0/1/2` 的 11 位串，以降低非手机号误伤。

---

## 与 `frontend.py` 其它逻辑的关系

- **Speaker 标签**：行首 `<|spk_…|>` 会先剥离，对正文做 TN，再拼回第一段（见 `frontend.py`）。
- **SSML 启发式**：正文中若仍含 `<|` 与 `|>`（剥离 spk 后），仍会 **整段跳过** wetext 前端；客服预 TN 已在 `text_normalize` 靠前位置对含中文文本执行，与是否跳过 wetext 独立，需结合业务避免在正文插入其它 `<|…|>`。

## 维护建议

- 新增银行/运营商号码：在 `_rules_carrier_hotline` 中增加 `_mk2(品牌正则, "号码")`。
- 新增句式：优先加在 **更具体的规则函数** 内；兜底手机规则放最后。
- 修改读法（幺 vs 一）：调整对应 `_serial_zh(..., yao_one=...)`。
- 详细正则以 `customer_service_tn.py` 为准；本文档随大版本迭代同步更新。
